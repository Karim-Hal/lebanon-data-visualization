# Lebanon Crisis Dashboard — HTML Report Design Spec
**Date:** 2026-04-30  
**Authors:** Reem Marji & Karim Hallal  
**Status:** Approved

---

## 1. Goal

Replace the Streamlit multipage app with a single, self-contained `report.html` generated by a Python script (`generate_report.py`). The report preserves all existing Plotly visualizations, applies the project's canonical color palette and design tokens, and presents them in a scrollable editorial layout where each section is a dense dashboard panel (not a chart-per-row list).

---

## 2. Architecture

### Generator approach

`generate_report.py` (project root):
- Imports all existing `src/` modules (data_loader, metrics, config) without modification.
- Reads the 6 CSVs + GeoJSON directly.
- Builds every Plotly figure using the same logic already in `pages/`.
- Serializes each figure to JSON via `fig.to_json()`.
- Renders a single Jinja2-style (or f-string) HTML template and writes `report.html`.

`report.html`:
- Loads Plotly.js from CDN (`https://cdn.plot.ly/plotly-2.35.2.min.js`).
- Embeds all figure JSON as `<script type="application/json">` blocks.
- All other JS/CSS is inline (no external dependencies beyond Plotly CDN).
- Embeds Google Fonts: **DM Serif Display** (section titles) + **DM Sans** (body, labels, nav).

### Data flow

```
CSVs + GeoJSON  →  generate_report.py  →  report.html
     (src/)              (Python)             (static)
```

No server needed. `report.html` opens directly in a browser.

## 4. Layout & Sections

### Global chrome

- **Sticky top nav** (height: 52px, deep-navy background): site title left, 5 anchor links right (Landing · Macro Shock · Food Prices · Food Insecurity · Health Toll). Active section highlighted via IntersectionObserver JS.
- **Page background**: `#F8F9FA`.
- **Section separator**: 1px `#D0D9E8` line + section number badge.

### Section 0 — Landing (Hero)

Full-width dark panel (`deep_navy` background):
- Large display headline: "Lebanon in Crisis" in DM Serif Display, white, 56px.
- Subtitle paragraph, light opacity.
- **4 KPI cards** in a CSS grid row — border-left accent in `PALETTE[0]` (inflation, GDP) and `PALETTE[1]` (IPC), `PALETTE[3]` (LBP rate). Value in large bold DM Sans.
- **Crisis timeline** Plotly chart below cards (full width, 180px height).

### Section 1 — Act 1: The Macro Shock

Two-column layout where appropriate:
- **GDP + Official Rate dual-axis** — full width (height 420px).
- **Inflation small multiples** — full width, 6-facet grid (height 500px).
- **GDP per capita table** — full width, styled HTML table (not `st.dataframe` — native `<table>` with Lebanon row highlighted).

### Section 2 — Act 2: Food Price Transmission

- **Price index multi-line** — full width (height 420px). All basket items shown (no slicer needed — show all).
- **LBP vs USD dual-axis** — full width (height 420px).
- **Exchange rate scatter** — 55% width (left column), R² annotation box inset. Right 45%: a prose callout card explaining the correlation finding.

### Section 3 — Act 2 Extended: Who Suffers Most

- Row 1: **Choropleth** (60% width) + **population group donut** (40% width).
- Row 2: **IPC phase stacked bar** (65% width) + **governorate basket price bar** (35% width). Use most-recent snapshot date for all charts (no slicer — show latest).

### Section 4 — Act 3: The Health Toll

- **Lag panel** (food price index vs stunting, shared X, 2-row subplot) — full width.
- **2×2 nutrition small multiples** — full width.
- **Dumbbell chart** (before/after 2019) — full width.
- **MENA under-5 mortality line** — full width.

## 5. Plotly Figure Styling

Apply globally to every figure via `fig.update_layout()`:
```python
fig.update_layout(
    font=dict(family="DM Sans, sans-serif", color="#1F3864"),
    paper_bgcolor="#FFFFFF",
    plot_bgcolor="#FFFFFF",
    margin=dict(l=48, r=24, t=36, b=48),
)
```

Grid lines: `gridcolor="#EEF1F5"` on both axes.  
Event vlines: reuse existing `EVENTS` list from `config.py`.  
No changes to chart logic — only cosmetic layout/font overrides.
